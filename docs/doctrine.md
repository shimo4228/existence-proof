## 1. 一段落の主張

世界同時多発的に（仮説 — §2）、学位・所属・資格を持たない者が、AI によって初めて自身の特異な認知パターンを成果物として表出可能にし、これまで制度がゲートしていた種類の仕事を生み出し始めている。本戦略の主題はこの層の**エンパワーメント**である: 層に名前を与え（§3）、存在証明 — 「一人でも実例があれば、その類の存在は証明される」— と再現可能な playbook を attraction-based に供給する（§7）。著者の研究プログラム（5 research line）は本 doctrine の主題ではなく、命題「学位なしでも制度級の成果物は作れる」の**構成的証明 = worked example** である（§5）。供給された artifact が読まれ、読んだ者が自発的に言及するとき earned channel が生まれ、それは LLM citation の最大源（earned 48% > owned 23%）として authorship-strategy の owned-channel 戦略を外側から補完する（§5, §8）。能動的な promotion・勧誘・組織化は一切行わない（§6）。

## 2. 現象と仮説

### 定義（仮説）

**AI-powered humans**（仮称 — 命名は再オープン中、§3 と naming-candidates 参照）: 以下の3条件の交点にいる人々。

1. **社会的位置**: 産出する成果物の領域で、その産出を制度がゲートしてきた credential（学位・所属・資格・職業訓練）を持たず、当該領域の評価系から周縁化されてきた。credential 不在は**成果物領域に相対的**に判定する — 全方位的に credential を持たない「持たざる者」だけでなく、他領域の職業経験は持つが成果物領域の gating credential を欠く「非コーダーのドメイン専門家」型（[habitat-map §6](research/2026-06-05_habitat-map.md) で観測）も条件を満たす
2. **認知特異性**: 特異な認知パターン・関心・意欲を持つが、それを社会的成果物に変換する手段（訓練・資本・組織）にアクセスできなかった
3. **AI enabler**: LLM をはじめとする AI によって、初めてその潜在能力を制度級の成果物（研究・ソフトウェア・著作）として表出できるようになった

「**制度級**」の定義（固定 — 2026-06-05、habitat-map Caveat 3 の伸縮を防ぐ）: 従来その産出に制度的訓練・所属・資格が事実上前提とされてきた**類**の成果物で、第三者が検証可能な外部アンカー（live deploy・解決する DOI・実ユーザーへの納品 等）を伴うもの。成果物の「類」に関する主張であり、規模・品質・human peer review 通過の主張ではない。

### 認識論的規律（load-bearing）

- Phase 0 の生息地検証（2026-06-05、[habitat-map](research/2026-06-05_habitat-map.md)）により、「**その類の人々が存在する**」ことは観測で支持された — 外部アンカー（live deploy・解決する DOI・納品実績）付きの独立した一人称事例が ja / en 両圏で複数確認された
- ただし「世界同時多発的な**層としての規模・分布**」は依然**仮説**である。観測は n=1 存在証明の集積であり、定量的シェア・増加率ではない（habitat-map 結論の二層構造）。規模・分布に関する主張は定量検証まで仮説として表記し、「〜のはずだ」「〜と推定される」を「〜である」と書き換えない
- これは authorship-strategy の empirical 規約（"evidence of X" でなく "observation consistent with X"）と同じ規律の適用である

## 3. 命名と scope discipline

### 命名の状態: 再オープン（2026-06-05 — 一時確定「独立峰」を suspend）

「独立峰」は衝突チェック通過の上で一時確定したが、**著者の文中試用テスト**（文中で人を指したとき意図しない笑いが起きないか — 命名者自身が真顔で使い続けられるか）で suspend となった。固有用語は cosmetic ではなく **semantic signature**（authorship-strategy Layer 4 tactic 4）であり、recognition 軸（§4）の中核 tactic である。以下の bullet 群は suspend 時点の記録として保持する:

- **確定経緯**: plan Phase 1 で候補 13 件 × 2 round の衝突チェック + 同 register 追加生成を経て、人・現象メタファー枠の空白（引用句・ブランド・辞書義・グリフター捕捉すべてゼロ）と register（山岳の威厳 = hustle 語彙から最遠）を確認の上、著者が確定。全候補の reject 理由は [research/2026-06-05_naming-candidates.md](research/2026-06-05_naming-candidates.md) が正本
- **運用**: 層・現象の**カテゴリ名**であって、自称を要求しない（層の自己記述は謙遜形 — habitat-map §4 — であり、峰の高さ含意は自称と衝突する）。初出文脈では必ず一行 gloss と共に導入する（bare token の初見は字義の山に読まれる）
- **agency の整合**: 独立峰の多くは成層火山 — 山脈の隆起（制度の引き上げ）ではなく自前の噴火で立った山。「AI が作った」読みを構造的に排除する
- **「存在証明」との分業**: 存在証明は term ではなく説明論理（gloss）として保持する（数理論理の witness — ∃x の立証実例 — が精密版）。repo 名 existence-proof との不整合はない
- **en term（2026-06-05 著者確定）**: **freestanding peak(s)**。衝突チェック結果: 人メタファー枠は空、Kilimanjaro（成層火山）binding はメタファーを補強、グリフター隣接なし — [naming-candidates en term チェック](research/2026-06-05_naming-candidates.md) 参照。運用規約: closed spelling 標準（free-standing は許容 variant）、**"peak" を必ず付ける**（bare 形 "the freestanding" / "freestanding creators" は家電 register に流れるため禁止形）。音訳 dokuritsuhō は使わない。inselberg / monadnock は語源補強の脚注素材として保持

**defensible scope**: 本概念の固有部分は「**周縁化されていた者の潜在的特異能力が、AI で初めて表出可能になる**」という社会的位置 × 認知特異性 × AI enabler の**交点**にある。遂行形態・作業様式・世代論はすべて prior art が豊富であり、そこで origin claim を立てない。claim は交点の命名に限定する。

### グリフター排除基準（segment 定義の負の境界）

「AI で稼ぐ」系の層は本概念の定義外である。判別基準:

- **制作物 narrative か、金銭 narrative か**: 本概念の層は「作れなかったものが作れるようになった」を語る。income claim（収益自慢・稼ぎ方の販売）を中心に置く自己呈示は segment 定義から排除する
- この境界は doctrine のすべての発信物に適用する。defining essay（§7 T1）はこの境界を本文で明示する

## 4. Audience-relative positioning — PBE 3次元の適用

> caveat: PBE（Personal Brand Equity: appeal / differentiation / recognition）は**個人**に対して validate された理論である（Gorbatov et al. 2021; MDPI 2025 — research/field-map §1）。研究プログラムおよび「層に向けた戦略」への適用は未検証であり、本節は理論の transfer 仮説である。

positioning は audience-relative である（PBE の定義は "in a given professional field"）。本 doctrine の primary audience は §2 の層であり、3次元はすべてこの audience 相対で定義する:

- **appeal（何が刺さるか）** = 存在証明 + playbook。この audience が価値を置くのは credential ではなく「**credential なしで実際にやれた証拠**」と「自分にも再現できる手順」である。伝統的アカデミア相対では減点要素（学位なし・所属なし・Zenodo deposit）が、この audience 相対では最大の価値に反転する
- **differentiation（何で他と違うか）** = DOI-registered idea-rescue + open diffusion + 再現可能な playbook の組合せ。同じ AI 文脈の発信物の中で、本プログラムは「何であるか」（検証可能なタイムスタンプ、open license、動く実装、評価系に依存しない蓄積）によって立つ。他の発信物への批判では立たない（§6 禁止事項）
- **recognition（何で思い出されるか）** = coined term 独立峰（§3）+ 既存固有用語群（AKC / Contemplative Agent / AAP / Authorship Strategy / Attention-Not-Self）。generic 語彙は paraphrase で消えるが、固有用語は token-level signal として残る

**open question**: 個人キャリア文脈で validate された3次元が、「層に向けたプログラム」文脈で十分かは未検証。固有の第4次元（例: reproducibility — playbook の再現成功率）が要るかは Phase 5 の再測定で再訪する。

## 5. 関係宣言 — 既存戦略との境界と接続

### 2×2 マップ（チャネルの座標系）

|  | on-page（自分の surface） | off-page（他者の surface） |
|---|---|---|
| **human** | `readme-writer`（README 人間ファースト） | **existence-proof（本 doctrine）** |
| **AI** | `llms-txt-writer`（llms.txt / llms-full.txt） | `authorship-strategy`（LLM-mediated diffusion） |

4象限のうち off-page × human だけが既存戦略・toolchain で未カバーだった。本 doctrine はこの1象限のみを固有領域とし、他の3象限を侵さない。

### authorship-strategy との関係: 補完（同一プログラム・別 audience・別チャネル）

- authorship-strategy の primary audience は LLM-mediated channels（LLM 直接 + LLM 経由の人間）。owned channel（repo / DOI / dataset mirror）への構造化投資で diffusion を最大化する
- existence-proof の primary audience は独立峰の層（人間）。彼らへの artifact 供給自体が目的（empowerment）であり、その副次効果として彼らの自発的言及 = earned channel が生まれる
- deep-research の確定 finding（research/field-map §3）: LLM citation の plurality（48%）は earned media であり、owned は 23%。つまり**earned channel は「人間向けの何か」ではなく、LLM citation の最大源**である。本 doctrine が機能するとき、人間への empowerment と authorship-strategy の LLM-mediated reach は同じ動きで強化される — 両戦略は同一 audience に別チャネルから収束する補完関係にある

### ADR-0007 との整合（load-bearing）

authorship-strategy [ADR-0007](https://github.com/shimo4228/authorship-strategy)（2026-05-29, v0.2.0）は「platform human-attention signal（stars / page-views）を success metric にしない。off-page human-distribution（被リンク獲得・告知）は red-ocean として投資しない」と決定済みである。本 doctrine はこの決定を**覆さない**:

1. 被リンク獲得活動・告知活動は本 doctrine でも禁止である（§6 attraction-based 限定、§7 除外リスト）
2. ADR-0007 が棄却したのは「**diffusion の手段としての** off-page human 投資」。本 doctrine の主題は diffusion ではなく **audience の empowerment** であり、人間に届くことは手段ではなく目的の構成要素である（empowerment は届かなければ成立しない）
3. 測定は stars / page-views を使わない（§8。ADR-0007 と同じ理由: gameable / LLM-mediated reach に盲目）
4. earned 48% finding は ADR-0007 リリース後に得られた事後証拠であり、「earned channel は human-attention play ではなく LLM citation の最大源」という新しい事実を加える。これは ADR-0007 の決定範囲（能動 distribution への投資判断）の外側にある

### 研究プログラム = worked example

authorship-strategy Layer 3 の「abstract doctrine + worked implementation ペア」構造を踏襲する。本 doctrine が abstract 側、著者の研究プログラム（5 line + 運用 harness）が worked 側である。「学位なしでも制度級の成果物は作れる」という命題に対し、プログラムの存在自体が構成的証明として機能する。doctrine 単独では「概念はわかるが自分にできるか分からない」、プログラム単独では「すごいが原理が読み取れない」— ペアで初めて audience の再現（= 彼ら自身の existence proof の追加）が起きる。

## 6. Not a movement — authenticity 緊張の設計課題化

### 原則: name and serve, not lead and recruit

本戦略は現象に**名前を与え**、存在証明と playbook を**供給する**。それ以上のことはしない:

- 組織化しない。community を運営しない。membership を作らない
- 勧誘しない。「あなたも〜すべきだ」と呼びかけない
- 広がりは読者の**自己同定**（「これは自分のことだ」）としてのみ起きる。それを起こすのは artifact の質であって、こちらの働きかけではない

この原則は attraction-based 制約（§7）の上位にある。movement 化は最も能動的な promotion であり、本戦略が採りうる最大の逸脱である。

### authenticity 緊張とその構造的最小化

personal branding は「戦略的な自己呈示」と「authentic self の表出」の間に本質的緊張を抱える（Gershon 2016 — research/field-map §2）。本 doctrine はこの緊張を否定せず、構造で最小化する:

- 著者は**実際に**この層の一人である。「自分が実際にそうである層に向けて、実際にやったことを示す」構造では、演出が原理的に不要になる — 示すべきものはすべて既に存在する（repo, DOI, 実装, 履歴）
- それでも残る緊張（どの事実を選んで見せるか、は依然として戦略的選択である）は、cloaking 禁止規約で抑える: **事実は全 surface で一致させる。梱包（見出し・詳細度・言語）は audience-driven に変えてよいが、主張は変えない**

### 禁止事項

authorship-strategy の禁止事項を継承した上で、本 doctrine 固有の禁止を加える:

- **継承**: マネタイズ提案（スポンサー / 有料 tier / コンサル化 / 収益化書籍）/ 競合批判・排他的 positioning / 売れるためのメッセージ調整 / バズ目的のセンセーショナル framing / origin claim の scope 過拡張
- **固有**: 人格ブランディング（著者個人を売る framing — idea と playbook が主語、著者は従）/ 勧誘・recruiting 言語 / membership・community 運営の提案 / グリフター隣接 framing（income claim、「AI で人生逆転」系の煽り）/ 層の規模・実在の断定（Phase 0 検証前 — §2）

## 7. Tactics — attraction-based の earned-channel 構築

### 概念の前提: earned は直接作れない

- **earned media** = 他者が自発的に書く media（editorial / forum / blog / SNS 言及）。これは定義上、こちらの行為では直接生成できない
- Zenn / Substack への自己投稿は **owned/rented** であって earned ではない。この区別を doctrine 内で崩さない
- 因果は2段階である: `[owned/rented で artifact を供給（制御可能）] → [人間が読む] → [読んだ者が自発的に言及（制御不能）] → [earned 化] → [LLM citation 源]`
- 本 doctrine の tactics はすべて**左2段（制御可能領域）の設計**である。右2段を直接操作する tactic（PR 依頼、被リンク獲得、相互言及）は存在しない — それは attraction-based の放棄であり、Skip 確定 list 違反である

### T1: defining essay（第1号 artifact）

coined term（§3）を定義する essay を書く。layer の存在仮説・3条件の交点・グリフター境界・存在証明の論理を、当事者の一人称で articulate する。

- 役割: recognition 軸の起点（term の初出）+ segment 検証装置（自己同定反応の観測 — §8 secondary metric）+ audience が共有したくなる第一の artifact
- 実装: `writing-ecosystem`（essay-reviewer / fact-checker）で執筆、Zenn（ja）→ Substack（en, `ja-to-en-translation` 経由）
- ガード: 勧誘で終わらない（「あなたもなれる」ではなく「あなたはすでにそうかもしれない」の記述まで）。income claim ゼロ

### T2: playbook artifacts（再現可能性の供給）

「どうやったか」を audience が再現できる粒度で人間向けに梱包する。既存の repo / skill / ADR / 運用記録が原料であり、新規制作物は梱包だけである。

- ガード: 自慢にしない（worked example の提示は「できた」の記録であって「すごいだろう」の演出ではない）。再現手順は clone + copy で済む形に（friction minimization 継承）

### T3: rented channel 発信（人間が読む入口）

`writing-ecosystem` / `substack-publishing` / `collect-context` の既存 pipeline で、note（ja）+ Zenn（ja）+ Substack（en）に T1 / T2 系の記事を出す。

- **venue 設計（2026-06-05 改訂。スケール戦略の正本: [ADR-0001](adr/0001-japanese-first-scale-strategy.md)）**: T1 defining essay は **note (ja・独立峰 line 専用新規アカウント) 初出 + Substack (en) 準同時。Zenn は後続/参照 hub**。理由: (a) 層のジャンル適合は note（一人称 essay 文化。habitat 最強事例は note 発）、(b) en 言説（vibe coding / 報道）で交点が先に命名される時間窓リスク → en を line 昇格まで遅らせない、(c) Zenn は engineer 寄りで「非エンジニア」層の人口構成とずれるが、既存 cadence の後続記事からの参照で hub として機能させる
- 位置づけ: これは owned/rented であり earned ではない（上記前提）。ただし「人間が読む」段を成立させる唯一の制御可能な入口である
- ガード: 発信 ≠ 売り込み。記事は artifact の共有であって、リーチ獲得の運用（投稿時刻最適化・ハッシュタグ戦略・バズ狙いタイトル）はしない。zenn-content 側の house rule（バズタイム投稿）より本ガードが優先される

### T4: README 人間ファースト化（着地点の整備 — Part B 吸収）

human-channel SEO direction の Part B（hub + 5 line repo の README を `readme-writer` で人間ファースト化 + repo メタ整備）を本 doctrine の tactic として実行する（plan Phase 3）。

- why の供給: audience が独立峰の層と定まったことで、「誰向けに README を書くか」が初めて確定した。earned 言及から辿ってきた人間が着地するのは README である
- 制約継承: 「README は hygiene であり growth lever ではない」（human-channel memory / ADR-0007）。過剰投資しない。事実の正本は human-channel memory、本 doctrine は引用のみ

### T5: semantic signature の点検（既存資産の branding 観点レビュー）

固有用語群が llms.txt / graph.jsonld に semantic signature として載っているかを点検する（plan Phase 2）。新規生成はしない — authorship-strategy の既存 tactic の点検のみ。

### 明示的除外（Skip 確定 list + 本 doctrine 固有）

awesome-list への PR / Wikidata 自己登録 / GitHub Pages 人間 LP / Wayback 能動登録 / preprint server（Zenodo 以外）/ 被リンク獲得活動 / 相互言及・相互紹介の打診 / インフルエンサーへの送付 / 勧誘

## 8. 測定

### primary: citation source mix

regurgitation test（LLM に概念を問い、応答と引用源を観測する — authorship-strategy の既存手法）を**source 分類に拡張**する: 引用源を owned（自 repo / DOI / dataset）/ earned(第三者の記事・forum・言及) / commercial に分類し、earned 比率の時系列変化を追う。

- baseline（2026-06-05 時点）: earned mention ≈ 0。clone:view ≈ 16:1（2026-05-29 実測、human-channel memory）— 人間到達がほぼゼロの状態から始まる
- 期待される動き: T1-T4 が機能すれば、earned 比率が方向として増える。絶対値の目標は置かない（gameable 化を避ける）

### secondary: self-identification signal

- 「これは自分のことだ」型の反応（コメント・引用・リプライ）
- 第三者による言及・派生（彼ら自身の existence proof 記事の出現が最強の signal — creative reuse の本 doctrine 版）

### 除外する指標

GitHub stars / page-views（ADR-0007: gameable / LLM-mediated reach に盲目）。follower 数・SNS インプレッション（同様に gameable で、勧誘 incentive を生む）

### 認識論的 caveat

LLM-mediated reach は原理的に不可視であり（human-channel memory）、source mix の観測も probe した LLM・時点に依存する。測定は方向性の確認であって、効果の証明ではない。empirical 規約（§2）に従い "observation consistent with X" の tone で記録する。
